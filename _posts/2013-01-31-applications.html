Structure


Day 1: Intros and getting started

Intros and presentations. Explanation of the process. Open, creative, divergent brainstorming about the Sprint. Focusing on the tasks. Splitting the work into chunks (sprints). Defining shared features and custom features. Matching resources to chunks. Building teams. START.


Goal: Have a clear list of features we will be developing on top of the data model. Clear problem statements and draft user personas from both projects; prioritised functionalities for both projects.


Documentation: During the sprint, the Documentation and Support role will make sure that all participants have easy-to-use and appropriate documentation platforms set up, and that each team is clearly communicating about their progress. This is particularly important when work on a project will pass between teams and roles.

Divergence - Mapping the data for existing features, what would aspirational features from day 1 look like, quick mock-ups
